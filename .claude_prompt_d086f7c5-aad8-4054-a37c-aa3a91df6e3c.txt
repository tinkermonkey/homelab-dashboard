**Title**:
[PR Feedback] Error Handling & Degradation Signaling

**Description**:
Based on consolidated PR review, address the following in **Error Handling & Degradation Signaling**:

- **Silent fallback to mock data masks production failures**: `mcp-transformer.ts:11-18` and `:22-35` catch MCP errors and silently fall back to hardcoded mock data. The server sends HTTP 206 with a `degraded` array, but `useAPI.ts:8` only throws on `!response.ok` — 206 is a success status, so the degradation signal is completely lost. The frontend never knows it is showing fabricated data. An operator sees green dashboards while actual infrastructure could be down. `server/src/transformers/mcp-transformer.ts:11-18`, `client/src/hooks/useAPI.ts:8`
- **NtopNG client silently returns zeros on every failure**: Three methods in `ntopng-client.ts` use bare `catch {}` with no logging: `getInterfaceData()` (line 64), `getDNSStats()` (line 92), `getVpnPeers()` (line 103). When ntopng is unreachable, the dashboard shows zeros as if there is no activity. `server/src/clients/ntopng-client.ts:61-105`
- **ElastiFlow client silently returns zero throughput**: `getCurrentThroughput()` at `elastiflow-client.ts:99-101` catches all errors and returns `{ down: 0, up: 0 }` with no logging. Bare `catch {}` discards errors entirely. `server/src/clients/elastiflow-client.ts:99-101`
- **Server route handlers have no error handling**: `/api/status`, `/api/cluster`, `/api/docker`, `/api/topology` routes in `index.ts:70-126` have zero try-catch protection. `getCachedData` (lines 51-67) also has no error handling — rejected promises propagate with no logging or wrapping. `server/src/index.ts:51-126`
- **No React error boundary in the application**: `main.tsx` and `App.tsx` have no error boundary. Any component render exception crashes the entire application to a white screen with no recovery. `client/src/main.tsx:15`, `client/src/App.tsx`
- **Overview error state indistinguishable from loading state**: `App.tsx:231-234` renders the same `PlaceholderView` for loading, error, and no-data states. The error object from React Query is completely discarded. Users see "Coming in Phase 2+" when the API fails. `client/src/App.tsx:231-234`
