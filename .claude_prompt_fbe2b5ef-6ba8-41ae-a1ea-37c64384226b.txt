**Title**:
Phase 5: Topology View Replacement with Heimdall Graph Components

**Description**:
Replace the six custom topology files (~519 lines total) with Heimdall's graph rendering engine. This is the highest-risk migration due to the fundamental difference between the current hardcoded absolute positioning model and GraphCanvas's force-directed layout. The architecture decision is to use fixed initial positions derived from the current layout while gaining zoom/pan and bezier edge rendering.

## Requirements
- Map `TopologyBot[]` and delegation relationships into `GraphCanvas` node and edge data structures
- Use fixed initial node positions derived from current `TP_BOT_LAYOUT` and `TP_COL_X` constants (per ADR-3 — do NOT use force-directed layout for initial migration)
- Replace `BotCard` with Heimdall's `TopologyNode`, mapping `bot.status`, `bot.avatar`, and `bot.label` to node props
- Replace `TopologyEdges` with `GraphEdge` + `bezierPath`/`rectEdgePoint` utilities for edge rendering
- Replace `BotInspector` with `GraphInspector`, mapping bot metadata, key-value properties, MCP list, manages list, and delegates list
- Replace `TopologyStage` with `GraphCanvas`, enabling zoom and pan
- Delete custom topology files: `TopologyStage.tsx`, `TopologyEdges.tsx`, `BotCard.tsx`, `BotInspector.tsx`, `ServerHeader.tsx`, `TopologyLegend.tsx`
- `PageHeader` from Phase 3 must already be in place for the Topology view header
- `selectedTopologyBot` persisted UI state must continue to control the selected/inspected node

## Design Guidance
Implements Phase 5 (Tier 2 Structural Replacements — Topology) from the architecture design and follows ADR-3 (Topology Rendering — Force Layout vs. Fixed Positioning). The Shared Type System established pattern governs how `TopologyBot` types map to graph node props.
