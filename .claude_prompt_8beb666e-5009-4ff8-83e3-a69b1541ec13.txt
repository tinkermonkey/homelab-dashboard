Based on consolidated PR review, address the following in **Chat & Bot Console**:

- **XSS via `dangerouslySetInnerHTML` on chat messages**: `ChatRail.tsx:62` renders message body HTML via `dangerouslySetInnerHTML={{ __html: b.p }}`. Server-sourced bot responses arriving via the `/api/chat/:botId` SSE proxy are rendered as raw HTML with no sanitization — a stored XSS vector. Fix: sanitize all HTML with DOMPurify before rendering, or switch to safe React element rendering. `client/src/components/chat/ChatRail.tsx:62`
- **SSE chat client integration not wired up**: The server correctly implements SSE streaming via `POST /api/chat/:botId`, but `ChatRail.tsx` makes zero network calls — it uses a hardcoded `replies` lookup table with canned strings per bot ID, storing results in local React state only. The architecture specifies end-to-end SSE transport and the server half is built, but the client half is entirely absent.
- **Duplicate diverging type definitions for chat types**: `shared/src/index.ts:75-104` and `client/src/components/chat/types.ts:1-36` define separate `Bot`, `ThreadMessage`, `ThreadItem` hierarchies. `ChatRail.tsx` imports from the local copy while API data comes typed from shared. These are structurally compatible only by coincidence and will diverge. `shared/src/index.ts:75-104`, `client/src/components/chat/types.ts:1-36`
